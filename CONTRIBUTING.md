# Contributing

このプロジェクトは基本的に個人開発ですが、将来的なコラボレーションを想定して最低限のルールをまとめています。

1. **チケット化** – 変更の背景と完了条件を Issue にまとめてから手を動かしてください。
2. **ブランチ運用** – `feature/<topic>` や `fix/<topic>` の枝で作業し、小さなコミットに分割します。
3. **検証** – `pytest backend/tests`、`npm run test -- tests/schemas.test.ts`、`terraform plan` など影響範囲に応じたチェックを実行し、結果を PR に添付します。
4. **ドキュメント** – README に記載している仕様・運用・ポリシーに影響する場合は、同 README の該当セクション（アーキテクチャ / デプロイ手順 / ポリシーなど）を更新してください。

命名規則や RSS/LLM の扱いも README 内で説明しています。迷ったときはまず README を開く、という運用にしています。
